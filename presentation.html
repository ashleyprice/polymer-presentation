Polymer Elements & Material Design
Polymer: Everything is elements
Polymer is a low level UI library that makes building apps easier.
Polymer is a LIBRARY not a FRAMEWORK.
They consider the DOM the framework.
Return to early days of HTML: UI is made in HTML and manipulated using DOM APIs
Polymer team is part of the chrome team. In the past, building frameworks that are above the platform (use the web platform as a rendering layer).
What is Polymer
Polymer is a web platform thing (not a chrome thing). Web components are a new standard.
Polymer is built on Web Components
Composable with Shadow DOM. Consumable with HTML
Polyfills aren't polymer. They're written by the polymer team to let polymer apps run on other recent browsers.
Polyfills lets this work across a number of browsers
Polymer is a thin layer on top of web components. Web components are primative types. Polymer (1) Reduce boilerplate (2) Introduces a declarative syntax
Leveraging Polymer
Polymer has a set of Core Elements. Pull down the code for the element you want and import it
Defining & Composing custom elements super easy:
template will capture the element's "view" put a style in the template template is rendered in the Shadow DOM hidden from other elements stopping style code from leaking into other areas of your DOM. Add a script Polymer comes with two way data binding with mustache syntax. Polymer comes with event binding.
What Polymer is good for: Building apps made of elements. Building elements out of elements Building sets of elements to pass around to folks
